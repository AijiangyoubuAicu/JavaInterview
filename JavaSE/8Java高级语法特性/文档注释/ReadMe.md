# 文档注释

## 概述
JDK包含了一个有用的工具`javadoc`，它是可以由源文件生成一个 HTML 文档；

如果在源代码中添加以专用的定界符`/**`开始的注释，那么可以很容易地生成一个看上去很有
专业水准的文档，这是一个很好的方式，因为这种方式可以将代码和注释宝尊在同一个地方；如果将文档存入一个独立的文件中，随着时间的推移，
出现代码和注释不一致的问题，然而文档注释与源代码在同一文件中，在修改源代码的同时，重新运行`javadoc`就可以轻易保持两者的一致性；

## 注释的插入：

javadoc 实现程序从下面几个特性中抽取信息：
- 包
- 公有类与接口
- 公有的和受保护的域特性

应该为上面几部分编写注释，注释应该放置在所描述特性的前面；注释以`/**`开始，以`*/`结束；

每个/**..*/文档注释在标记之后紧跟着文本说明，标记由`@`开始，如"@author"或"@param"

文本说明的第一句应该是一个**概要性**的句子，javadoc会自动将这些句子抽取出来形成概要页；

文本说明可以使用 HTML 语言修饰；

## 类注释
类注释必须放在 import语句之后，类定义之前；

下面是一个类注释的例子： 

```java
/**
 * 这个 {@code Car} 类是一只很可爱的动物
 */
public class Car {
    
}
```
> 没有必要每一行用型号 * ，少点星号的话IDE会自动添加

## 方法注释
每一个方法必须放在所描述的方法之前，除了通用标记之外，还可以使用下面的标记：
- `@param` 变量描述 

这个标记将对当前方法的**参数列表**部分生成一个条目，这个描述可以占据多行，并可以使用HTML描述 一个方法的所有`@param`必须放在一起

- `@return` 返回值描述

这个标记将对当前方法添加**返回值**部分，这个描述可以跨越多行，并可以使用HTML标记；

- `@throws` 异常描述

这个标记将添加一个注释，用于表示这个方法有可能抛出的**异常**

## 域注释

只需对**公有域**(通常指静态常量)建立文档


事例代码：
```java
/**
 * 这个{@code Car}类是一种可爱的动物
 */
class Car {

    private Object Car = null;

    /**
     * 这是一个公有域，通常指静态常量
     * 只需为类属性建立文档
     */
    public static int count = 1;

    /**
     * 得到一只猫的方法
     * @param name 猫的名字
     * @param age 猫的年龄
     * @return 返回一只猫
     * @throws NullPointerException 空指针异常
     */
    public Car getCar(String name, int age) throws NullPointerException {

        return (Car) Car;
    }
}
```

## 通用注释

以下的标记可以用在文档注释当中：

- @author 代码作者名

这个标记将产生一个**author**(作者)条目，可以使用多个 @author 标记，每个 @author
对应一个作者

- @version 版本声明

这个标记将产生一个**version**(版本)条目，这里的文本可以是对当前版本的任何描述

- @since 引入条目的描述

这个标记将产生一个**since**(始于)条目，这里的text可以是对引入特性的版本的描述，如

@since version 1.7.1

- @deprecated 取代建议

这个标记将对类、方法或变量添加一个**不再使用**的注释，文本中给出取代的建议

- @see 引用

这个标记将在**see also**部分增加一个超链接，它可用于类中，也可以用于方法中，用法如：

    @see com.aijiang.dao.User#getUsername() (这个链接将定位于当前包或当前类中，后面的#号用于定位到方法或属性) 
    @see <a href="www.baidu.com"> (超链接到指定的URL)

## 包与概述注释

可以直接将类、方法和遍历的注释放置在Java源文件中，只要用`/**..*/`文档注释界定即可；
但是要想产生包注释，就需要每一个包目录中添加一个单独的文件，有以下两个选择： 

1. 提供一个以 package.html 命名的 HTML
   文件，在标记<body></body>之间的文本会被抽取出来<br> 
   
2.提供一个以 package-info.java 命名的 Java
文件，这个文件必须包含一个初始的以`/**..*/`界定的`javadoc`注释，跟随在一个包语言中，它不应该包含
更多的代码或注释

还可以为所有源文件提供一个概述性的注释，这个注释被放置在一个名为`overview.html`的文件中，这个文件
位于包含所有源文件的父目录中，标记<body></body>之间的所有文本将被抽取出来
